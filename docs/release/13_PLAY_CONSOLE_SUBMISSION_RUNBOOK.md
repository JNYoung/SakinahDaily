# Play Console Submission Runbook

Status: Launch-prep draft. Use this as the canonical Play Console submission
pack before creating the first closed-testing release.

Last checked against Google Play Help on 2026-06-11.

## Purpose

This runbook is the Play Console submission pack for Sakinah Daily. It turns
the release materials into one auditable checklist covering the app record,
Main store listing, App content declarations, release artifact, closed-testing
release, tester links, and the later Production access evidence path.

The local template gate is:

```sh
scripts/export_google_play_public_links_packet.sh
scripts/verify_google_play_public_links_packet.sh
scripts/export_google_analytics_debugview_packet.sh
scripts/export_google_play_upload_packet.sh
scripts/verify_google_play_submission_pack.sh
scripts/verify_google_play_production_access_pack.sh
scripts/export_google_play_production_access_packet.sh
scripts/verify_google_play_closed_test_launch_day.sh
scripts/export_google_play_closed_test_retention_packet.sh
```

The strict gate, after the human Play Console fields are complete, is:

```sh
SAKINAH_REQUIRE_PLAY_SUBMISSION_READY=true \
SAKINAH_PLAY_CONSOLE_APP_CREATED=true \
SAKINAH_PLAY_MAIN_STORE_LISTING_READY=true \
SAKINAH_PLAY_APP_ACCESS_READY=true \
SAKINAH_PLAY_ADS_DECLARATION_READY=true \
SAKINAH_PLAY_CONTENT_RATING_READY=true \
SAKINAH_PLAY_TARGET_AUDIENCE_READY=true \
SAKINAH_PLAY_DATA_SAFETY_READY=true \
SAKINAH_PLAY_PRIVACY_POLICY_READY=true \
SAKINAH_PLAY_STORE_SETTINGS_READY=true \
SAKINAH_PLAY_RELEASE_NOTES_READY=true \
SAKINAH_PLAY_SCREENSHOTS_READY=true \
SAKINAH_PLAY_FEATURE_GRAPHIC_READY=true \
SAKINAH_PLAY_CLOSED_TEST_RELEASE_DRAFTED=true \
SAKINAH_PRIVACY_POLICY_URL=https://privacy.sakinahdaily.app/privacy \
SAKINAH_PLAY_TESTING_FEEDBACK=support@sakinahdaily.app \
SAKINAH_PLAY_TESTER_GROUP_CREATED=true \
SAKINAH_PLAY_CLOSED_TRACK_READY=true \
SAKINAH_PLAY_APP_CONTENT_READY=true \
SAKINAH_PLAY_STORE_LISTING_READY=true \
scripts/verify_google_play_submission_pack.sh
```

The strict command also requires upload signing through `android/key.properties`
or the `SAKINAH_UPLOAD_*` environment variables because it delegates to
`scripts/verify_google_play_upload_preflight.sh`.

## Main store listing

Source: `docs/release/04_STORE_METADATA_DRAFT.md`.

- App name: Sakinah Daily.
- Short description: use the English listing candidate first:
  "Prayer times and local reminders, calm and private."
- Full description: use the policy-safe prayer-first copy from the metadata
  draft.
- Category candidate: Lifestyle. Confirm the final category and app tags in
  Play Console during store review.
- Feature graphic: use `build/store-assets/google-play-feature-graphic.png`,
  generated by `scripts/generate_google_play_feature_graphic.py` and verified
  by `scripts/verify_google_play_store_assets.sh`.
- Phone screenshots: use the Android screenshot matrix from
  `docs/release/05_SCREENSHOT_PLAN.md`.
- Keep listing claims aligned with the app: no ads, no tracking, no GPS
  permission in v0.1, no AI fatwa, no religious Q&A, no generic Quran TTS, and
  no background music under Quran recitation.

## App content

Complete these Play Console App content sections before the first submission:

- App access: no account or login is required for the v0.1 release path.
- Ads: declare no ads, because the app contains no ad SDK and no ad surfaces.
- Content rating: answer from the current app behavior. The MVP has religious
  content, no user-generated content, no social features, no gambling, no
  violence, no explicit content, and no medical period-tracking claims.
- Target audience and content: Sakinah Daily is not designed for children. The
  owner should choose the final age target in Play Console and keep the store
  copy prayer-companion focused.
- Data safety: use `docs/privacy/04_GOOGLE_PLAY_DATA_SAFETY_DRAFT.md` and
  `docs/privacy/06_SDK_AND_API_INVENTORY.md` as the source of truth. The v0.1
  app has no account, no ads, no crash SDK, no GPS permission, default-off
  Firebase Analytics, and local-first sensitive women's-mode state.
- Privacy policy: host the reviewed copy from
  `docs/privacy/02_PRIVACY_POLICY_DRAFT.md`, then pass the final URL through
  `SAKINAH_PRIVACY_POLICY_URL`.
- Public links hosting packet: prepare `build/play-public-links` with
  `scripts/export_google_play_public_links_packet.sh`, then publish the
  reviewed privacy page and optional HTTPS feedback page before strict upload
  preflight. Run `scripts/verify_google_play_public_links_packet.sh` before
  publishing to catch missing sections, placeholder copy, and feedback forms
  that could collect sensitive tester details.
- Google Analytics DebugView packet: prepare
  `build/google-analytics-debugview` with
  `scripts/export_google_analytics_debugview_packet.sh` before any reviewed
  analytics QA build transmits telemetry.
- Store settings: confirm category, contact email, website/privacy links,
  countries/regions, and app availability before release review.

## Closed testing release

Source: `docs/release/09_GOOGLE_PLAY_CLOSED_TESTING.md`.

- Build a signed AAB with `scripts/verify_google_play_internal_release.sh`.
- Run `scripts/verify_google_play_upload_preflight.sh` before upload.
  The preflight verifies the current AAB checksum and readable App Bundle
  zip/base manifest structure before the artifact is treated as upload-ready.
- Export `build/play-upload` with
  `scripts/export_google_play_upload_packet.sh` for final local review before
  creating or submitting the Closed testing release.
- Add `sakinah-daily-testers@googlegroups.com` to the Closed testing track.
- Configure Testing feedback with the same final email or URL supplied through
  `SAKINAH_PLAY_TESTING_FEEDBACK`.
- Confirm the in-app Closed testing guide shows copyable Day 1 / Day 3 /
  Day 7 / Day 14 feedback templates, local feedback-sent checkboxes, and the
  privacy reminder before sharing tester links.
- Use release notes from `docs/release/08_VERSION_AND_RELEASE_NOTES.md`.
- Submit all pending changes in Publishing overview.
- Share the tester group link first, then the Play testing opt-in link.
- After the closed-testing release is approved or visible to testers, run
  `scripts/verify_google_play_closed_test_launch_day.sh`.
- Share the Google Group link first, then the Play opt-in link; do not share
  the leave-testing link as an invite.
- Export the retention observation packet with
  `scripts/export_google_play_closed_test_retention_packet.sh` so Day 1 /
  Day 3 / Day 7 / Day 14 feedback themes can be summarized without tester
  personal data.
- Export the Day 0 / Day 1 operator packet with
  `scripts/export_google_play_day0_day1_operator_packet.sh` before sharing
  invite links and again before the first-use feedback review, so share order,
  evidence-log updates, Day 1 theme intake, and optional DebugView decisions
  are handled from one aggregate-only handoff.

## Production access

Sources:

- `docs/release/12_CLOSED_TESTING_EVIDENCE_LOG.md`.
- `docs/release/14_PRODUCTION_ACCESS_ANSWER_DRAFT.md`.
- `build/play-retention-observation/production_access_feedback_summary.md`
  after running `scripts/export_google_play_closed_test_retention_packet.sh`.

- Keep at least 12 opted-in testers for 14 continuous days if the account is
  subject to Google Play's testing requirement.
- Track aggregate counts only. Do not store tester personal data.
- Review Google Play Testing feedback and the configured feedback channel.
- Record feedback themes, changes made, and the Production access answer draft.
- Fill `production_access_feedback_summary.md` with aggregate Day 1 / Day 3 /
  Day 7 / Day 14 themes first, then copy the reviewed wording into the Play
  Console answer draft.
- Run `scripts/verify_google_play_closed_testing_evidence.sh` during launch
  prep and strict mode after the real 14-day test.
- Run `scripts/verify_google_play_production_access_pack.sh` in template mode
  before launch prep, then strict mode after the real 14-day closed test,
  feedback summary, changes summary, readiness evidence, and human answer
  review are complete.
- Export `build/play-production-access` with
  `scripts/export_google_play_production_access_packet.sh` before final human
  review, then rerun it in strict mode after the real closed-test evidence and
  Production access answer review are complete.
